# 🚀 GitHubへの手動デプロイ手順

PowerShellでエラーが発生する場合、以下の手順を**コマンドプロンプト（cmd）**で実行してください。

## ステップ1: コマンドプロンプトを開く

1. Windowsキー + R を押す
2. `cmd` と入力してEnter
3. プロジェクトフォルダに移動：
   ```
   cd C:\Users\FMV\Desktop\Q-Quest_量子神託
   ```

## ステップ2: リモート設定を確認

```
git remote -v
```

**もし `<YOUR_USERNAME>/<YOUR_REPO_NAME>` と表示される場合**、ステップ3に進んでください。

**もし何も表示されない場合**、ステップ4に進んでください。

## ステップ3: リモートURLを修正（必要な場合）

既存のリモートを削除：
```
git remote remove origin
```

正しいURLでリモートを追加（以下を実際のURLに置き換えてください）：
```
git remote add origin https://github.com/あなたのユーザー名/リポジトリ名.git
```

**例**:
```
git remote add origin https://github.com/tanaka/Q-Quest-Quantum-Oracle.git
```

## ステップ4: GitHubリポジトリを作成（まだの場合）

1. [GitHub](https://github.com) にアクセス
2. 右上の「**+**」→「**New repository**」をクリック
3. リポジトリ名を入力（例：`Q-Quest-Quantum-Oracle`）
4. 「**Create repository**」をクリック
5. 表示されるURLをコピー（例：`https://github.com/your-username/Q-Quest-Quantum-Oracle.git`）

## ステップ5: ファイルをコミット

```
git add .
git commit -m "Update: Q-Quest 量子神託 with 絵馬納め機能"
```

## ステップ6: ブランチをmainに設定

```
git branch -M main
```

## ステップ7: GitHubにプッシュ

```
git push -u origin main
```

初回はブラウザでGitHubの認証が求められる場合があります。

## ステップ8: Streamlit Cloudでデプロイ

1. [Streamlit Cloud](https://streamlit.io/cloud) にアクセス
2. 「**Sign in**」→「**New app**」をクリック
3. リポジトリを選択
4. **Branch**: `main`
5. **Main file**: `app.py`
6. 「**Deploy**」をクリック

---

## トラブルシューティング

### エラー: `fatal: unable to access 'https://github.com/...'`

→ リモートURLが正しくないか、リポジトリが存在しません。ステップ3または4を確認してください。

### エラー: `fatal: The current branch main has no upstream branch.`

→ ステップ7の `git push -u origin main` を実行してください（`-u` オプションが重要です）。

### 認証エラー

→ GitHubの認証が必要です。ブラウザで認証画面が開くので、ログインしてください。
